# Provider Dashboard — Release Notes v2.5.0

**Release date:** April 2026

**Applies to:** Provider Dashboard (all staff logins)

This document matches the in-app **What’s new** release notes (v2.5.0 popup). The popup uses icons and short blurbs for each theme below.

---

## Overview

This release focuses on **Settings**, the **treatment plan** experience, how **leads** appear alongside your clients, and **status icons** on each client row in the list.

---

## What’s new (high level)

### Settings hub — notifications & treatment pricing

Notifications and treatment pricing each have their own page under Settings, with live previews and **Request changes** when you need updates from support.

### Section icons on the client list

Each row shows three icons for **Plan**, **Analysis**, and **Quiz** (in that order). A **check** means that area is complete or on track (for example: plan has items, quiz finished, analysis ready). A **minus** means not started, or no plan items yet. Facial analysis can show a **clock** while it’s still pending.

### Unified edit panel for each treatment

Optional details (quantity, product options, areas, notes, and similar) are edited in **one panel** with labelled sections, so only the fields that apply to the selected treatment are shown.

### Skincare quiz on the treatment recommender

Client quiz results (skin type and sensitivities) stay visible above treatment suggestions while you build a plan.

### Leads on their own screen

Treatment Finder leads use a dedicated **Leads** view, separate from the in-clinic client list.
